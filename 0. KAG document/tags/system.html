系統操作標籤

腳本閱讀器系統相關內容，亦是Adventure Game的系統共用操作，在本系統中主要為時間軸管理。

等待時間控制：
本系統的等待時間是一個基準值，亦指文字與文字中間顯示的時間差。
這樣設計是為了使畫面顯示的動態與文字播放對齊，但考量並非所有媒體皆需要對齊，因此各別的動態時間則交由標籤內額外設定。

  • delay
    指定字間延遲時間
  • wait
    指定一個等待時間
  • waitclick
    點擊等待
  • wc
    指定一個等待時間，時間長度由文字數量決定
  • autowc
    自動調整字符顯示速度，時間長度由文字數量決定

閱讀器操作：
閱讀器行為操作，動態時間省略、閱讀器操控、閱讀模式(省略點擊等待)。

  • s
    停止；停止閱讀器
  • clickskip
    允許啟用『跳過』
  • nextskip
    允許啟用『移動到下個選項』；與『跳過』，但直到選項出現才停止『跳過』
  • cancelskip
    停止『跳過』
  • automode
    允許啟用『自動閱讀』
  • cancelautomode
    停止『自動閱讀』

主系統介面資訊：( 功能確認，尚不需實做 )
系統在顯示上所有的共通資訊。

  • title
    設定標題
  • cursor
    變更滑鼠游標
  • rclick
    右鍵設定

外部資源載入：( 功能確認，尚不需實做 )
系統在顯示上所需要外部提供的資源。

  • loadplugin
    載入plug-in
  • mappfont
    載入字型圖檔

automode (允許啟用『自動閱讀』)

屬性必須數值說明
enabled yes true(Default) | false 自動閱讀功能若啟動設為true，不啟動設為false。
使用者使用『自動閱讀』功能時，系統是否運行此像運作。
自動閱讀為系統功能，此功能目的在讓閱讀器忽略點擊等待，在動態完成時直接顯示後續的內容。
停止自動閱讀，請使用cancelautomode

autowc (自動調整字符顯示速度，時間長度由文字數量決定)

屬性必須數值說明
enabled no true(default) | false 使否啟用這像功能。
ch no 目標字符 自動調整等待時間的指定字符。
time no 等待時間(文字數量) 指定字符各自的等待時間，其等待時間長度由文字數量乘上單一文字延遲時間(delay)。
指定特定字符的文字延遲時間。
當文字顯示到ch屬性內所指定的字符，其文字延遲時間即為time內所指定的對應時間。
(20150311 註解) 原則上此功能指定字符的延遲時間會覆蓋delay的設定，所以當『跳過』功能實行時，其狀況與一般文字相同。
例如：
[autowc enabled=true ch="、。・" time="3,5,3"]

cancelautomode (停止『自動閱讀』)

強制解除『自動閱讀』功能。
若後續內容不允許閱讀器使用自動閱讀來瀏覽內容，使用此功能暫停。
注意，將automode設為Disabled並不能停止自動『自動閱讀』，只能不再啟用該功能。
例如：
00:[automode enabled=true]
↑ 允許自動閱讀
01:It is content 1.[l][r]
↑ 使用者觸發自動閱讀，從01到03行的點擊等待皆忽略
02:It is content 2.[l][r]
03:It is content 3.[l][r]
04:[cancelautomode]
↑ 停止自動閱讀，從05到06行的點擊等待不被忽略
05:It is content 4.[l][r]
06:It is content 5.[l][r]

cancelskip (停止『跳過』)

強制解除『跳過』功能。
『跳過』功能是指當文字瀏覽的字間延遲、動態表演、點擊等待時，使用者若不想花費時間，則以點擊為訊號跳過此時間。
當前系統只有nextskip的『移動到下個選項』功能才會啟動自動『跳過』，其於標籤為要求等待，因此本功能為解除『移動到下個選項』行為。
注意，將nextskip設為Disabled並不能停止自動『跳過』，只能不再啟用該功能。
例如：
00:[nextskip enabled=true]
↑ 允許移動到下個選項
01:It is content 1.[l][r]
↑ 使用者觸發移動到下個選項，從01到03行的點擊等待與延遲皆忽略
02:It is content 2.[l][r]
03:It is content 3.[l][r]
04:[cancelskip]
↑ 停止自動跳過，從05到06行的點擊等待與延遲不被忽略
05:It is content 4.[l][r]
06:It is content 5.[l][r]

clickskip (允許啟用『跳過』)

屬性必須數值說明
enabled yes true(Default) | false 跳過(Click Skip，點擊跳過)功能若啟動設為true，不啟動設為false。
點擊跳過，亦指點擊非元件單元時，觸發系統點擊訊號，並以此跳過文字顯示延遲、動態、點擊等待。

nextskip (允許啟用『移動到下個選項』)

屬性必須數值說明
enabled yes true(Default) | false 『移動到下個選項』功能若啟動設為true，不啟動設為false。
使用者使用『移動到下個選項』功能時，系統是否運行此像運作。
(20150311 註解) 因為移動為快進功能，因此當啟用此功能，同時也啟用『跳過』。 (20150519 註解) 移動到下個選項為系統功能，此功能目的在讓閱讀器忽略一切動態，直到下個選項出現。

delay (指定字間延遲時間)

屬性必須數值說明
speed yes "nowait" | "user" | 數值(毫秒、ms) "nowait"，延遲速度為0，每個文字皆不停頓的逐一顯示。
"user"，延遲速度依據[System|文字速度]所設定的預設速度 ( 設定來源 ? )。
數值，延遲速度依據設定之毫秒(ms)時間逐一顯示文字。
文字顯速度度設定後，其後的所有內容皆受此影響，直到下次改變。
指定文字顯示時，字符間延遲的速度。
例如：
[delay speed=1000]
xyz
顯示結果：
  1. x
  2. [延遲1秒]
  3. xy
  4. [延遲1秒]
  5. xyz

  • click
  • timeout
  • link

wait (指定一個等待時間)

屬性必須數值說明
time yes 數值(毫秒、ms) 依據設定的毫秒(ms)時間等待。
mode no "normal"(default) | "until" 指定等待的方式。
"normal"，依據time所指定的時間等待。
"until"
canskip no true(default) | false 當滑鼠、鍵盤敲下，這標籤是否接受『跳過』功能；若接受則為true、不接受則為false。
等待一段文字數量長度的延遲時間。
例如：
[wait speed=5000 canskip=true]
結果：
[等待5秒]

點擊等待。
不同於l標籤、p標籤，不接受『跳過』功能，會顯示等待圖示(符號)。

wc (指定一個等待時間，時間長度由文字數量決定)

屬性必須數值說明
time yes 數值(文字數量) 等待時間長度由設定之數值乘上單一文字延遲時間。(delay)
canskip no true(default) | false (20130311 註解)未確保標籤功能統一，新增此屬性。
當滑鼠、鍵盤敲下，這標籤是否接受『跳過』功能；若接受則為true、不接受則為false。
等待一段文字數量長度的延遲時間。
例如：
[delay speed=1000] [wc time=5]
結果：
[等待5秒]
